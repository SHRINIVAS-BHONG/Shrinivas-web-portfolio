How I work

Development process

A straightforward process, in this order, for every project — so you always know what's happening and what's next.

01

Discovery

Understanding your goals, users, and constraints before writing a single line of code — so the technical decisions actually serve the project.

02

Planning & structure

Mapping the site or app's information architecture, key flows, and technical approach, so scope and direction are clear from the start.

03

Design

Designing interfaces in Figma — or refining an existing design — with attention to typography, spacing, and interaction detail.

04

Development

Building with React, Next.js, and Tailwind CSS, with clean, componentised code that's easy to maintain and extend later.

05

Testing & review

Checking responsiveness, performance, and accessibility across devices, and reviewing the build together before launch.

06

Launch & support

Deploying to production on Vercel, then staying available for fixes, updates, and improvements after the site goes live.

About

I build websites and web applications for clients who care about quality — the kind of work that's fast, clean, and holds up under real use.

My approach is straightforward: understand the problem before proposing a solution, keep the codebase clean enough that anyone can pick it up later, and treat every project — big or small — with the same level of care.

I work primarily with React, Next.js, and Tailwind CSS on the frontend, and Python with FastAPI when a project needs a custom backend. Every build is version-controlled, deployed on Vercel, and tested across devices before it ships.
